Source diversity (age groups, genders, locations)

3. Security & Privacy Architecture
Encryption:

At Rest: AES-256-GCM for all databases
In Transit: TLS 1.3 for all API communication
Evidence Vault: Client-side encryption, keys never leave user's device
Password Storage: bcrypt with cost factor 12

Anonymization by Default:

PII stripped before storage
IP addresses immediately hashed (SHA-256)
Location detection at district-level only (no GPS precision)
User identity never linked to reports (optional accounts use separate storage)

Blockchain Audit Trail:

Evidence hashes + timestamps stored on Polygon (low gas fees)
Immutable proof of when evidence was collected
Smart contract verification: verifyEvidence(hash) ‚Üí (exists, timestamp, reportId)

User-Controlled Sharing:

Default: Reports and evidence completely private
Authorities: Optional one-time shareable link (expires in 7 days)
Public: Never shared without explicit user consent

DDoS Protection:

Cloudflare in front of API Gateway (absorbs up to 100 Gbps attacks)
Rate limiting: 100 requests/minute per IP
Challenge-response for suspicious traffic

Compliance:

GDPR-inspired: Data minimization, right to erasure, user consent
ISO 27001 principles: Information security management

Bug bounty program for ethical hackers


4. Failure Handling & Reliability
Graceful Degradation:

Intelligence engine fails ‚Üí Reports still accepted (processed from queue later)
Blockchain unavailable ‚Üí Evidence stored locally, logged when service restored
SMS gateway down ‚Üí Push notifications still sent, SMS retried every 5 minutes

Circuit Breaker Pattern:

Timeout: 3 seconds for external API calls
Error threshold: 50% failures triggers circuit open
Reset timeout: 30 seconds before retry
Fallback: "Service temporarily unavailable. Using cached data."

Database Failover:

PostgreSQL: Read replica auto-promoted to master if primary fails (<2 min recovery, zero data loss)
MongoDB: Replica sets with automatic failover
Redis: Cluster mode with failover to replica nodes

Multi-Region Disaster Recovery:

Primary region: Mumbai (AWS ap-south-1)
Backup region: Singapore (AWS ap-southeast-1)
RPO (Recovery Point Objective): 15 minutes max data loss
RTO (Recovery Time Objective): 30 minutes to full restoration

Monitoring & Alerting:

Prometheus + Grafana dashboards tracking: Request rate, error rate, latency (p50/p95/p99), database connections, cache hit rate, queue depth
PagerDuty/Opsgenie alerts: Error rate >5% for 5 min, latency >2 sec, disk >80% full
